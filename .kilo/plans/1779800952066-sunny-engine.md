## 3. High-Level System Architecture

**Guiding Principle**: **Zero-Trust + Layered Defense**. LLM/orchestrator = untrusted advisor. Guardian = trusted (but limited) executor. Data = validated + provenance-tagged. All decisions = immutable, queryable, replayable.

### Core Layers (Strict Boundaries)

**1. Data Plane (Trusted Ingestion)**
- Primary: Indodax Market Data WS (tickers, trades, orderbook, kline) + Private WS (real fills, order lifecycle).
- Fallback/History: REST + Trade API v2.
- Storage: Timescale hypertables (OHLCV, trades, features) + Redis hot cache + pgvector for semantic.
- Normalization + validation + sanity bounds (e.g. price spikes >10% in 1m flagged).
- Future: On-chain (public RPCs for whale flows), alt data (with heavy sanitization).

**2. Intelligence / Cognition Plane (Untrusted Reasoning)**
- **Orchestrator** (enhance existing): Persistent sessions (save state to DB), multi-turn context, HITL approval nodes, topological waves + retries.
- **Specialized Agents** (via registry):
  - QuantAlpha (current tools + enhanced).
  - Risk & Compliance Guardian (independent veto — always consulted).
  - Research/Narrative (deep analysis, post-mortems).
  - Execution Planner (proposes only; never executes).
- **Decision Engine**: Multi-factor fusion (SMC + TA + narrative + whale/meme + AI consensus + regime ML + uncertainty score) → Predator-style scoring → position sizing (fractional Kelly / CVaR / volatility target) → confidence-gated.
- **Memory**: Tiered + typed
  - Episodic: Individual trades (with full context, outcome, lessons).
  - Semantic: Market regimes, pair behaviors, strategy efficacy.
  - Procedural: Versioned strategy configs/prompts (with backtest results attached).
  - Forgetting/importance: Decay low-utility memories; periodic audit + "nuke" snapshots.
- All LLM calls: Logged with prompt hash, inputs, raw output, parsed decision, cost.

**3. Execution Plane (Hardened, Auditable Boundary — Most Critical)**
- **TradeGuardian** (new dedicated service/process, separate runtime from LLM):
  - Policy engine (hard rules first, later OPA/Cedar): max notional/day, per-pair exposure, daily loss kill, correlation limits, min confidence, max slippage, cooldowns, recvWindow enforcement.
  - Pre-trade: Balance recon (vs exchange), full risk check (reuse + extend RiskManager), kill switch query.
  - Execution: Only via scoped Indodax keys (spot-only, no withdraw, IP if possible). Supports limit/market, client_order_id, partial fill handling.
  - Post-trade: Immediate private WS verification + reconciliation; immutable ledger entry.
  - Sandbox: Docker/container with strict egress (only indodax.com + LLM provider domains). No shared secrets with cognition plane.
  - Dual-mode: Paper (current) + Live (gated).
- Never allow direct `execute_trade` from orchestrator/QuantAgent in live mode.

**4. Control & Governance Plane**
- **Scheduler**: Event-driven (WS triggers) + time-based (BullMQ repeatable). Multiple strategies.
- **Monitoring & Alerting**: Prom + Grafana (equity curves, risk heatmaps, decision funnel, token burn), Telegram bot (trade notifs, anomalies, approval requests, kill alerts).
- **Audit & Compliance**: Append-only `decision_audit` + `trade_ledger` tables (full context, hashes, signatures if needed). Export for tax (Indonesia crypto rules).
- **Strategy Registry**: Versioned (prompts, params, risk config, backtest metadata). A/B + shadow testing.
- **Human Oversight**: Dashboard + TG for approvals, manual override, strategy kill.

**5. Cross-Cutting**
- **Error Handling & Resilience**: Circuit breakers, graceful degradation (paper fallback), auto-recon on restart.
- **Security**: Least privilege everywhere, key rotation policy, input sanitization + provenance on all external data, memory poisoning detection.
- **Scalability Hooks**: Queue partitioning (by pair/strategy), read replicas, feature store.

**Trade-offs & Alternatives**:
- Custom StateGraph vs LangGraph.js: **Custom wins** (auditability, no black-box, lower attack surface, full control over money paths). Use LangGraph only for non-critical experiments.
- Full autonomy vs HITL: **Mandatory HITL** for live trades above threshold or until 500+ validated paper trades + proven edge. Full auto only in paper shadow + low-AUM guarded mode.
- Monolith vs microservices: Start monolith + separate guardian process; split when team/scale demands.
- Pure TS vs Python sidecar: Hybrid (TS core + Python for heavy ML/BT libs) — best of both. Comm via Redis/BullMQ.

**Current Code Alignment**: Excellent base. Prioritize: WS clients, audit logging, guardian extraction, Timescale hypertables for performance, persistent orchestrator state.
